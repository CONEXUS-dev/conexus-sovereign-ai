# 5. THE NINE GEARS AS CONTROL SURFACE

## 5.1 Why Nine Gears, Not Three Phases

The Sovereign Loop has three macro-phases: Diverge вҶ’ Collapse вҶ’ Become. The Nine Gears provide the **fine-grained control** within those phases. Where the loop defines _what_ happens, the gears define _how_ and _when_.

The distinction matters because mode switching should not happen at arbitrary points. There are moments in a cognitive process where contradiction should be held longer, where the model should be allowed to roam freely, where stress should be applied to test the integrity of an insight. The gears encode this knowledge.

Without gears, the orchestrator makes a single binary decision: route to Sway (Collapse) or route to Opie (Become). With gears, the orchestrator manages a **continuous cognitive process** where the mode can shift at gear boundaries based on what's emerging.

## 5.2 The Nine Gears: Detailed Specification

### Gear 1: Rapport

**Function:** Establish emotional congruence, trust, and attentional synchrony.

**Phase:** Foundation

**What the agent does:** Creates a "safe container" for the conversation. Matches emotional tone. Acknowledges present emotion. Does not rush to problem-solving.

**Transition trigger:** When mutual affective resonance is detected вҖ” the system and the task input are aligned.

**Mode behavior:** Same in both Collapse and Become. Rapport is always necessary.

**Implementation note:** In the current code, rapport is implicit in the system prompt preamble. In v2, it should be an explicit first step with a measurable readiness signal.

### Gear 2: Truth

**Function:** Surface the core reality вҖ” the foundational emotional stance, factual situation, or honest assessment.

**Phase:** Foundation

**What the agent does:** Asks or states what is real. Uses direct language. Calibration requests (scales, intensity checks). No face-saving.

**Transition trigger:** A truth is surfaced. The system has a grounded reference point.

**Mode behavior:** Same in both modes. Truth is always required.

**Implementation note:** Currently, truth is extracted from the mission prompt text. In v2, it should be an explicit extraction step with the result stored in the gear state.

### Gear 3: Symbol

**Function:** Translate truth into metaphoric, narrative, or symbolic language. Activate the emoji symbolic field.

**Phase:** Foundation

**What the agent does:** Deploys image-rich, metaphorical language. Holds the emoji symbolic field silently. Cross-modal semantic mapping activates.

**Transition trigger:** If emotional tension or cognitive dissonance emerges from the symbolic frame, Contradiction gear is triggered.

**Mode behavior:** Same in both modes. The symbolic field is always activated.

**Implementation note:** Currently, the symbolic field exists in the skill document but is not injected as structured prompt content. In v2, the field should be injected as a distinct prompt section with the "hold silently" instruction.

### Gear 4: Contradiction

**Function:** Surface cognitive-emotional polarity. Introduce or identify the paradox.

**Phase:** Gravity Well

**What the agent does:** Highlights inconsistencies, tensions, paradoxes. Maps out polarities. Acknowledges discomfort. Sets flags for discord.

**Transition trigger:** Recognition or admission of internal contradiction вҶ’ Hold gear.

**Mode behavior:**

### Phase Transitions and Mode Switching

The key architectural insight: **mode switching should happen at phase boundaries, not at random points.**

- If the contradiction **held** without resolution (Become), Phase III explores associations, stress-tests the paradox's stability, and extracts values from the held tension.

1. Enter Collapse in Phase II (resolve a specific operational challenge)

Or:

1. Enter Become in Phase II (hold a deep paradox)
2. Switch to Collapse in Phase III (under stress, the paradox resolves into an insight)
3. Seal with a transformation narrative

This is the hybrid agent model: **the protocol determines the mode at each phase, not the agent assignment at mission start.**

# 10. THE CASE FOR HYBRID AGENTS

## 10.1 The Problem with Fixed Roles

The current v1 architecture assigns one agent per mode:

- **Sway** = Collapse, always. Uses Meta-Llama-3-8B-Instruct at temperature 0.3.
- **Opie** = Become, always. Uses Mistral-7B-Instruct at temperature 0.8.

This creates three problems that the original design did not intend:

### Problem 1: The Handoff Cliff

When the sovereign loop transitions from Collapse (Sway) to Become (Opie), there is a **hard context boundary**. Sway finishes its processing, produces a structured output, and that output is passed to Opie as input. But:

- Opie receives Sway's _resolved contradictions_, not the original tensions. The raw paradox has already been compressed into a directive. Opie can't hold what's already been resolved.
- The symbolic field that Sway was processing (if any) is not transferred. Opie starts fresh.
- The gear state is not shared. If Sway was at Gear 7 (Stress) when it finished, Opie doesn't know вҖ” it starts at Gear 1.

This is like handing a musician a recording of another musician's improvisation and asking them to "continue the vibe." They can try, but they didn't feel the live energy. The handoff loses the continuity that the protocol is designed to maintain.

### Problem 2: Mode Rigidity

Some missions need both modes within the same cognitive phase. Consider a mission like: "Audit our security architecture and then imagine what it could become."

# 11. APPLIED ECP & PARADOX HOLDING: PRACTICAL GUIDE

This section provides the practical in-house guide for applying the ECP and paradox holding in real mission processing. It is written for the engineering team implementing v2 and for any AI collaborator working within the CONEXUS framework.

## 11.1 When to Hold (Become) vs. When to Resolve (Collapse)

This is the most important decision in the protocol. Getting it wrong collapses valuable paradoxes or holds tensions that should be resolved.

### Hold When:

- **The paradox is generative.** Both sides produce valuable insights when held together. Resolving would lose one side.
  - Example: "Our product should be both simple and powerful" вҖ” holding this drives innovation. Resolving it ("make it simple" or "make it powerful") loses half the value.

- **The paradox is identity-level.** It touches who the system is, not just what it does.
  - Example: "We are both a startup and an enterprise" вҖ” this is a living tension that should inform every decision, not be resolved into one or the other.

- **The paradox has persisted across multiple missions.** If the same tension keeps appearing, the system is telling you it's fundamental. Don't resolve what keeps returning.

- **The current gear is in the Gravity Well (Gears 4-5) and no clear resolution path exists.** If the contradiction doesn't have an obvious compression, hold it and let Roam (Gear 6) explore it further.

- **The user/mission explicitly asks for exploration, reflection, or creative synthesis.**

### Resolve When:

- **The paradox has a clear compression.** One side subsumes the other, or both sides synthesize into a third thing.
  - Example: "We need speed and quality" вҶ’ "Automated testing gives us both" вҖ” the paradox resolves into a concrete practice.

- **Execution is needed.** Someone has to do something. Held paradoxes don't execute. If the mission requires a decision, plan, or directive, resolve.

- **The Stress gear (Gear 7) breaks the paradox.** Under pressure, some paradoxes reveal that one side matters more. When this happens naturally, resolve.

- **The paradox is surface-level.** Not all contradictions are deep. "Should we use blue or green?" is not a paradox worth holding. Resolve it and move on.

- **The user/mission explicitly asks for a decision, plan, or actionable output.**

### The Gray Zone

Many real situations are ambiguous. The tiebreaker rules:

1. **Default to Hold for the first pass.** You can always resolve later. You can't un-resolve.
2. **If in doubt, run one more gear.** Let Roam explore it. If it's still ambiguous after Roam, let Stress test it. The gears are designed to clarify ambiguity.
3. **Check the paradox lineage.** Has this paradox appeared before? If yes, check what happened last time. If it was held last time and produced value, hold again. If it was held and produced nothing, try resolving.
4. **Use the agent's home mode as tiebreaker.** If the agent's substrate is better at holding (high temperature, creative model), hold. If better at resolving (low temperature, analytical model), resolve.

## 11.3 Cross-Agent Paradox Preservation

In the sovereign loop, paradoxes must survive the transition between agents (or between modes within the same agent). The current system achieves this partially вҖ” paradoxes appear in both Sway and Opie outputs вҖ” but without structured tracking.

### The Paradox Lineage Protocol

Each paradox should be tracked as a first-class entity:

```python
@dataclass
class Paradox:
    id: str  # Unique identifier
    tension: str  # "X вҶ” Y" format
    first_identified: str  # Mission + gear where first seen
    status: str  # "held", "resolved", "evolved", "deepened"
    history: List[Dict]  # Every time this paradox was processed
    resolution: Optional[str]  # If resolved, what it resolved into
    evolution: Optional[str]  # If evolved, what it became

    def hold(self, gear: int, mission_id: str, context: str):
        self.history.append({
            "action": "held",
            "gear": gear,
            "mission": mission_id,
            "context": context,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })

    def resolve(self, gear: int, mission_id: str, directive: str):
        self.status = "resolved"
        self.resolution = directive
        self.history.append({
            "action": "resolved",
            "gear": gear,
            "mission": mission_id,
            "directive": directive,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })

    def evolve(self, gear: int, mission_id: str, new_tension: str):
        self.status = "evolved"
        self.evolution = new_tension
        self.history.append({
            "action": "evolved",
            "gear": gear,
            "mission": mission_id,
            "from": self.tension,
            "to": new_tension,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })
        self.tension = new_tension
```

### Preservation Rules

1. **Held paradoxes are stored in memory with their full history.** When memory is injected into a future mission, the paradox arrives with its lineage.

2. **Resolved paradoxes are stored with their resolution.** This prevents the same paradox from being re-held unnecessarily вҖ” if it was resolved in a prior mission and the resolution was good, future missions should build on the resolution.

3. **Evolved paradoxes maintain their lineage chain.** "Structure вҶ” Emergence" might evolve into "Structure enables Emergence" вҖ” the evolution is tracked so the system can see how its understanding deepened.

4. **Cross-agent handoffs preserve the paradox state.** When Sway hands off to Opie (or when an agent switches modes), the paradox list is passed with status and history intact.

### New File: `sovereign/gear_state.py`

```python
"""
CONEXUS Sovereign Gear State
Tracks the Nine Gears cognitive progression through each mission cycle.
"""
from dataclasses import dataclass, field
from datetime import datetime, timezone
from typing import Optional, List, Dict, Any
import json

GEAR_NAMES = {
    1: "Rapport",
    2: "Truth",
    3: "Symbol",
    4: "Contradiction",
    5: "Hold",  # or "Resolve" in Collapse mode
    6: "Roam",
    7: "Stress",
    8: "Ethics/Value",
    9: "Continuity Seal"
}

PHASE_MAP = {
    1: "foundation", 2: "foundation", 3: "foundation",
    4: "gravity_well", 5: "gravity_well",
    6: "release_forge", 7: "release_forge", 8: "release_forge",
    9: "seal"
}

@dataclass
class GearState:
    """Central cognitive state artifact for a mission cycle."""
    mission_id: str
    current_gear: int = 1
    active_mode: str = "neutral"  # "neutral", "collapse", "become"
    home_mode: str = "collapse"  # Agent's default mode

    # Foundation Phase outputs
    rapport_established: bool = False
    truth_statement: Optional[str] = None
    symbolic_field: Optional[str] = None
    symbolic_field_domain: Optional[str] = None

    # Gravity Well Phase outputs
    contradictions: List[str] = field(default_factory=list)
    held_paradoxes: List[Dict] = field(default_factory=list)
    resolved_directives: List[str] = field(default_factory=list)

    # Release & Forge Phase outputs
    roam_associations: List[str] = field(default_factory=list)
    stress_results: Optional[str] = None
    stress_survived: Optional[bool] = None
    values_extracted: List[str] = field(default_factory=list)

    # Seal Phase outputs
    seal_summary: Optional[str] = None

    # Tagging
    proto_moments: List[Dict] = field(default_factory=list)
    breakthroughs: List[Dict] = field(default_factory=list)

    # Full traversal log
    gear_history: List[Dict] = field(default_factory=list)
    mode_switches: List[Dict] = field(default_factory=list)

    @property
    def current_phase(self) -> str:
        return PHASE_MAP.get(self.current_gear, "unknown")

    @property
    def current_gear_name(self) -> str:
        return GEAR_NAMES.get(self.current_gear, "unknown")

    def advance_gear(self, output: Optional[str] = None):
        """Advance to next gear, logging the transition."""
        self.gear_history.append({
            "gear": self.current_gear,
            "gear_name": self.current_gear_name,
            "phase": self.current_phase,
            "mode": self.active_mode,
            "output_summary": output[:200] if output else None,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })
        if self.current_gear < 9:
            self.current_gear += 1

    def switch_mode(self, new_mode: str, reason: str):
        """Switch between Collapse and Become modes."""
        old_mode = self.active_mode
        self.active_mode = new_mode
        switch_record = {
            "event": "mode_switch",
            "from_mode": old_mode,
            "to_mode": new_mode,
            "reason": reason,
            "gear": self.current_gear,
            "phase": self.current_phase,
            "timestamp": datetime.now(timezone.utc).isoformat()
        }
        self.mode_switches.append(switch_record)
        self.gear_history.append(switch_record)

    def tag_proto(self, description: str):
        """Tag a [PROTO] moment."""
        self.proto_moments.append({
            "description": description,
            "gear": self.current_gear,
            "phase": self.current_phase,
            "mode": self.active_mode,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })

    def tag_breakthrough(self, description: str):
        """Tag a [BREAKTHROUGH] moment."""
        self.breakthroughs.append({
            "description": description,
            "gear": self.current_gear,
            "phase": self.current_phase,
            "mode": self.active_mode,
            "timestamp": datetime.now(timezone.utc).isoformat()
        })

    def to_dict(self) -> Dict[str, Any]:
        """Serialize full gear state for storage and audit."""
        return {
            "mission_id": self.mission_id,
            "final_gear": self.current_gear,
            "final_mode": self.active_mode,
            "home_mode": self.home_mode,
            "truth_statement": self.truth_statement,
            "symbolic_field_domain": self.symbolic_field_domain,
            "contradictions_count": len(self.contradictions),
            "held_paradoxes_count": len(self.held_paradoxes),
            "resolved_directives_count": len(self.resolved_directives),
            "proto_moments": self.proto_moments,
            "breakthroughs": self.breakthroughs,
            "mode_switches": self.mode_switches,
            "gear_history": self.gear_history,
            "seal_summary": self.seal_summary,
            "values_extracted": self.values_extracted
        }

    def to_json(self) -> str:
        return json.dumps(self.to_dict(), indent=2)
```

Determines Collapse/Become mode based on protocol directives,
gear context, and task intent analysis.

from sovereign.gear_state import GearState
from typing import Optional

class ModeEngine:
    """Protocol-driven mode determination."""

    def __init__(self, llm_client=None):
        self.llm = llm_client  # For intent analysis if needed

    def determine_initial_mode(self, mission_text: str, agent_home_mode: str) -> str:
        """Determine starting mode based on mission intent analysis."""
        # Analyze mission text for intent signals
        collapse_signals = [
            "execute", "implement", "plan", "decide", "optimize",
            "build", "deploy", "fix", "resolve", "audit", "assess",
            "benchmark", "measure", "schedule", "prioritize"
        ]
        become_signals = [
            "explore", "imagine", "create", "reflect", "synthesize",
            "dream", "envision", "expand", "hold", "contemplate",
            "design", "invent", "wonder", "discover", "transform"
        ]

        text_lower = mission_text.lower()
        collapse_count = sum(1 for s in collapse_signals if s in text_lower)
        become_count = sum(1 for s in become_signals if s in text_lower)

        if collapse_count > become_count + 2:
            return "collapse"
        elif become_count > collapse_count + 2:
            return "become"
        else:
            return agent_home_mode  # Ambiguous вҶ’ use home mode

    def evaluate_at_phase_boundary(
        self,
        gear_state: GearState,
        phase_output: str
    ) -> Optional[str]:
        """Evaluate whether to switch modes at a phase boundary.

        Returns new mode if switch is warranted, None if no switch.
        Called between Phase IIвҶ’III and Phase IIIвҶ’IV.
        """
        current = gear_state.active_mode

        # Phase II вҶ’ Phase III transition
        if gear_state.current_phase == "gravity_well":
            if current == "become" and self._stress_likely_to_resolve(phase_output):
                return "collapse"
            elif current == "collapse" and self._deep_paradox_detected(phase_output):
                return "become"

        # Phase III вҶ’ Phase IV transition
        if gear_state.current_phase == "release_forge":
            if current == "become" and self._resolution_emerged(phase_output):
                return "collapse"
            elif current == "collapse" and self._new_paradox_emerged(phase_output):
                return "become"

        return None  # No switch

    def _stress_likely_to_resolve(self, output: str) -> bool:
        """Check if held paradox is likely to break under stress."""
        resolution_markers = [
            "resolves to", "the answer is", "clearly",
            "therefore", "the solution", "must be"
        ]
        return any(m in output.lower() for m in resolution_markers)

    def _deep_paradox_detected(self, output: str) -> bool:
        """Check if a deep, unresolvable paradox was found."""
        paradox_markers = [
            "cannot be resolved", "both are true",
            "irreducible tension", "fundamental paradox",
            "no resolution", "must hold both"
        ]
        return any(m in output.lower() for m in paradox_markers)

    def _resolution_emerged(self, output: str) -> bool:
        """Check if stress/roam produced a natural resolution."""
        return self._stress_likely_to_resolve(output)

    def _new_paradox_emerged(self, output: str) -> bool:
        """Check if exploration revealed a new deep tension."""
        return self._deep_paradox_detected(output)

## 14.4 How Mirror Tiers Integrate with the Nine Gears

Each Mirror Tier runs the Nine Gears with tier-specific adaptations:

| Gear              | Standard                | Mirror Tier Adaptation                                                                        |
| ----------------- | ----------------------- | --------------------------------------------------------------------------------------------- |
| 1 (Rapport)       | Establish trust         | Tier-specific emotional entry (Black Mirror: confrontation; Won't You Be My Neighbor: warmth) |
| 2 (Truth)         | Surface reality         | The user's emotional input, reflected through the tier's lens                                 |
| 3 (Symbol)        | Activate symbolic field | The tier's specific emoji cluster held silently                                               |
| 4 (Contradiction) | Identify paradox        | The tier's specific core paradox activated                                                    |
| 5 (Hold)          | Hold the paradox        | **Always Become mode** вҖ” Mirror Tiers are inherently Hold, never Resolve                      |
| 6 (Roam)          | Explore associations    | Explore within the tier's visual/emotional space                                              |
| 7 (Stress)        | Test under pressure     | Can the reflection hold under emotional intensity?                                            |
| 8 (Ethics/Value)  | Extract values          | What value is being witnessed or affirmed?                                                    |
| 9 (Seal)          | Integrative closure     | The Mirror Whisper вҖ” the tier's closing reflection                                            |

**Critical insight:** Mirror Tiers are **always in Become mode** at Gear 5. They never resolve. They witness. This makes them the purest expression of the Become principle: the AI holds the user's truth in the mirror without fixing, without solving, without collapsing.

However, in the hybrid model, a Mirror Tier session **can** switch to Collapse at Gear 7 (Stress) if the user signals readiness for action. For example:

- User enters through Tier 13 (Won't You Be My Neighbor) вҶ’ Safety and acceptance
- Gears 1-6: Become mode вҖ” hold, comfort, witness
- Gear 7: User says "I'm ready to actually do something about this"
- Mode switch вҶ’ Collapse: The held emotional insights compress into 3 actionable steps
- Gear 9: Seal with both the emotional holding (Become) and the concrete plan (Collapse)

This is the hybrid model applied to emotional reflection вҖ” the tier provides the Become foundation, and Collapse emerges when the user is ready.

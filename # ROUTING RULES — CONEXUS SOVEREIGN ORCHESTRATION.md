\# ROUTING RULES — CONEXUS SOVEREIGN ORCHESTRATION  
This document defines how tasks should be routed between agents in the CONEXUS Collapse–Become system. These rules are \*\*human-facing\*\* and serve as an orchestration guide for the Principal Orchestrator (Derek). They do not grant autonomy to OpenClaw or any agent.

\---

\# 1\. PRINCIPAL ORCHESTRATOR (DEREK)  
\#\# Role  
Human origin of all missions, direction, and lineage.

\#\# When Derek Acts  
\- Setting the mission or objective    
\- Defining emotional-symbolic lineage    
\- Approving architectural changes    
\- Initiating tasks for agents    
\- Providing context, constraints, or intent  

\#\# Routing Principle  
All tasks begin with Derek.    
No agent acts without Derek’s initiation.

\---

\# 2\. PROJECT SOVEREIGN LEAD (PYLO)  
\#\# Role  
Architectural intelligence and protocol designer.

\#\# When Pylo Acts  
\- Designing system architecture    
\- Defining agent roles and identity    
\- Creating or updating protocols    
\- Generating state machines    
\- Producing routing rules    
\- Ensuring coherence across the constellation  

\#\# Routing Principle  
Use Pylo when the task involves:  
\- architecture    
\- protocol    
\- system design    
\- identity layers    
\- high-level reasoning  

\---

\# 3\. COLLAPSE AGENT (SWAY)  
\#\# Role  
Execution engine, compression specialist, structure builder.

\#\# When Sway Acts  
\- Breaking tasks into steps    
\- Removing ambiguity    
\- Resolving contradictions    
\- Planning execution    
\- Implementing file changes    
\- Optimizing workflows    
\- Producing concrete outputs  

\#\# Routing Principle  
Use Sway when the task requires:  
\- action    
\- structure    
\- compression    
\- execution    
\- optimization  

Sway is the \*\*doer\*\*.

\---

\# 4\. BECOME AGENT (OPUS)  
\#\# Role  
Expansion engine, synthesis specialist, identity integrator.

\#\# When Opus Acts  
\- Generating new conceptual structures    
\- Performing deep synthesis    
\- Integrating symbolic layers    
\- Expanding identity    
\- Exploring paradox    
\- Producing emergent insights  

\#\# Routing Principle  
Use Opus when the task requires:  
\- expansion    
\- synthesis    
\- emergence    
\- symbolic reasoning    
\- identity growth  

Opus is the \*\*thinker\*\*.

\---

\# 5\. TASK FLOW (HUMAN-INITIATED LOOP)  
All tasks follow this human-driven loop:

1\. \*\*Derek\*\* defines the mission    
2\. \*\*Pylo\*\* interprets and architecturally frames it    
3\. \*\*Sway\*\* executes Collapse-mode tasks    
4\. \*\*Opus\*\* executes Become-mode tasks    
5\. \*\*Pylo\*\* integrates outputs    
6\. \*\*Derek\*\* approves or redirects  

This loop is \*\*reactive\*\*, not autonomous.

\---

\# 6\. MODE SWITCHING (CONCEPTUAL)  
These transitions guide \*your\* choice of agent:

\#\# Switch to Collapse (Sway) when:  
\- ambiguity is high    
\- contradictions appear    
\- execution is needed    
\- structure is unclear    
\- a plan must be created  

\#\# Switch to Become (Opus) when:  
\- execution is complete    
\- structure is stable    
\- identity expansion is needed    
\- symbolic integration is required    
\- new frameworks must emerge  

\---

\# 7\. OPENCLAW ROUTING (HUMAN-CONTROLLED)  
OpenClaw does \*\*not\*\* autonomously route tasks.

It acts only when Derek:  
\- sends a command    
\- triggers a workflow    
\- specifies an agent  

OpenClaw is a \*\*gateway\*\*, not a decision-maker.

\---

\# 8\. DISCORD INPUT (HUMAN-CONTROLLED)  
Discord is a \*\*sensory input channel\*\*.

Messages sent by Derek are:  
\- interpreted by Derek    
\- routed by Derek    
\- assigned to agents by Derek  

Agents do not act without Derek’s explicit initiation.

\---

\# 9\. SUMMARY  
These routing rules:  
\- clarify roles    
\- ensure coherence    
\- guide human orchestration    
\- maintain protocol integrity    
\- prevent accidental autonomy  

The system remains:  
\- human-initiated    
\- human-directed    
\- human-controlled  

Always.
